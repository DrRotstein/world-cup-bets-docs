## Context

We use football-data.org v4 API (free tier, 10 req/min) as decided in ADR-001. Server-side caching avoids rate-limit issues. This story has no frontend — it's a backend data pipeline that feeds S-04 and S-05.

## Scope

- Match table: id, homeTeam, awayTeam, kickoffTime, status (scheduled/live/finished), homeScore, awayScore, matchday, group/stage
- Scheduled sync job (cron or interval): fetch fixtures + results from football-data.org
- Expose REST endpoint: GET /matches (list all), GET /matches/:id
- Filter matches by status, date, stage
- Cache responses to stay within rate limits
- Graceful error handling (retry on 429, log on failure, serve cached data)

## Non-goals

- No live minute-by-minute updates (we check every 5 min, not real-time websockets)
- No manual match entry (fallback for API outage is out of MVP scope; we accept the risk)
- No other competitions — World Cup 2026 only
